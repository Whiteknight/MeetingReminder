# Project Structure

## Solution: MeetingReminder.sln

### Projects

1. **MeetingReminder.Domain** (Core)
   - No dependencies
   - Contains: Entities, ValueObjects, Events, Errors

2. **MeetingReminder.Application** (Core)
   - Depends on: Domain
   - Contains: UseCases, Abstractions, Common

3. **MeetingReminder.Infrastructure** (Infrastructure)
   - Depends on: Application (→ Domain)
   - Contains: Configuration, Logging, Threading
   - Packages: System.Text.Json

4. **MeetingReminder.GoogleCalendar** (Vendor)
   - Depends on: Application (→ Domain)
   - Packages: Google.Apis.Calendar.v3

5. **MeetingReminder.ICal** (Vendor)
   - Depends on: Application (→ Domain)
   - Packages: Ical.Net

6. **MeetingReminder.Notifications** (Vendor)
   - Depends on: Application (→ Domain)
   - Contains: Beep, Flash, Sound, SystemNotification

7. **MeetingReminder.ConsoleTui** (Presentation)
   - Depends on: Infrastructure, GoogleCalendar, ICal, Notifications
   - Packages: Spectre.Console
   - Entry point for TUI frontend

## Naming Convention

- Base name: `MeetingReminder` (not `MeetingReminderTui`)
- Supports multiple frontends (TUI, GUI, Web, etc.)
- Frontend projects use descriptive suffixes (e.g., `ConsoleTui`, future: `WpfGui`, `BlazorWeb`)
